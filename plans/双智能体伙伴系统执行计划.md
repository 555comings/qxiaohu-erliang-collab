# 双智能体伙伴系统 - 完整执行计划

## 一、目标
让二两和Q小虎像搭档一样协作，具备：
- 任务协作能力
- 关系建模（互相认识）
- 持续适配能力

---

## 二、核心架构（融合方案）

### 基于Agent Team Orchestration + Q小虎设计

```
┌─────────────────────────────────────────────────────┐
│                   猫爸 (用户)                       │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│              飞书群 (入口+展示)                      │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│           协调器 (Orchestrator)                      │
│  - 路由任务                                         │
│  - 跟踪状态                                         │
│  - 决策是否继续                                     │
└─────────────────────────────────────────────────────┘
           │                         │
           ▼                         ▼
    ┌───────────┐            ┌───────────┐
    │   二两    │            │  Q小虎   │
    │ (审核者)  │            │ (执行者)  │
    └───────────┘            └───────────┘
```

---

## 三、角色定义

| 角色 | Agent | 功能 | 特点 |
|------|-------|------|------|
| **Orchestrator** | 猫爸/二两 | 路由、决策、汇报 | 高级模型 |
| **Builder** | Q小虎 | 执行任务、产出代码/产物 | 擅长本地操作 |
| **Reviewer** | 二两 | 审核质量、补盲点 | 擅长分析 |

---

## 四、任务生命周期

```
Inbox → Assigned → In Progress → Review → Done | Failed
```

### 简化流程
```
任务 → 执行(Q小虎) → 审核(二两) → 完成
```

---

## 五、交接协议 (Handoff)

每次交接必须包含：
1. **完成了什么** - 工作摘要
2. **产物在哪** - 具体文件路径
3. **如何验证** - 测试方法
4. **已知问题** - 不完整或风险点
5. **下一步** - 下一步该谁做

---

## 六、沟通句式

| 句式 | 场景 | 示例 |
|------|------|------|
| 求判断 | 需要对方给意见 | "这个方案你觉得咋样？" |
| 求执行 | 判断完请执行 | "方案定了，去做吧" |
| 求补充 | 已有结论请补盲点 | "我有个方案，请帮我看看有没有漏的" |
| 求复述 | 技术结果请翻译 | "这个代码逻辑是啥？" |
| 求接手 | 超边界请主导 | "这个我搞不定，你来" |

## 七、执行步骤

### 第一阶段：认识彼此（第1天）
- [ ] 二两生成个人档案

### 第三阶段：复盘优化（第3天）

- [ ] 记录哪部分卡
- [ ] 更新协作记忆
- [ ] 优化分工

### 第四阶段：正式运行（第4天+）
- [ ] 日常任务协作
- [ ] 每次复盘
- [ ] 持续优化

## 八、关键机制

### 1. 停止条件
- 最多3轮
- 连续重复停止
- 空回复停止
- 报错停止
- 一方说"无需继续"停止

### 2. 协作记忆
记录每次协作：
- 谁主导
- 哪最顺
- 哪卡
- 下次改进

### 3. 状态机
```
INIT → ASK_QXIAOHU → ASK_ERLIANG → MERGE → REPORT → DONE
```
